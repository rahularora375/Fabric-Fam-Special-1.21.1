# ROUTER.md

Task-to-file routing rules. When starting a task in this repo, match it against the rules below and load only the listed topic files. Default to loading fewer files; use `INDEX.md` as a fallback lookup when the task doesn't match any rule cleanly.

Always-loaded: `CLAUDE.md` (skeleton: project facts, working style, commands). Everything below is load-on-demand.

## Routing rules

- **Adding a new item, theme, or set bonus (status-effect half)** → `ITEMS.md`. If the new item touches `ArmorEffects.BONUSES` (any set-bonus status effect, helmet-flag bonus, HUD badge, durability regen, main-hand swap), also load `SYSTEMS.md`. If it adds a new `DataComponentType`, also load `SYSTEMS.md` (for `ModComponents`).
- **Adding attribute modifiers (+max health, speed, armor) to an armor piece** → `ITEMS.md`. No other file needed — values render natively in the vanilla attributes tooltip.
- **Attack-triggered behavior (heal-on-hit, debuff-on-hit, lifesteal, bleed)** → `ITEMS.md` + `SYSTEMS.md` (`AttackHandlers` section). If the effect must replace vanilla damage, use `AttackEntityCallback`; if additive, use `ALLOW_DAMAGE`. See the pattern note in `SYSTEMS.md`.
- **Editing or adding a mixin / gating vanilla screens / durability-cap work / enchant-pipeline interception** → `MIXINS.md`. If the mixin reads a component flag, skim `SYSTEMS.md` (`ModComponents` section) for the component's owner and other consumers.
- **Instant-apply / instant-strip dispatch (equip hook, slot hook, click-slot / creative-drag hook)** → `MIXINS.md` (`LivingEntityEquipMixin`, `ServerPlayNetworkHandlerMixin`, `ServerPlayNetworkHandlerClickSlotMixin`) + `SYSTEMS.md` (`ArmorEffects.refreshBonusesFor` helper). All three mixins call `ArmorEffects.refreshBonusesFor(player)` TAIL — one entrypoint, covers both APPLY and STRIP via the MOD_MANAGED diff. The 80-tick tick loop is the eventual-consistency safety net.
- **Custom status effect (registration, icon, HUD badge, particle override)** → `SYSTEMS.md` (`ModStatusEffects` section). If the effect is driven by a `Bonus` entry, also read the `ArmorEffects` section in the same file. Icons are byte-for-byte vanilla copies — do NOT author custom art.
- **Changing loot drop rates, tier weights, or map destinations** → `SYSTEMS.md` (Loot injection section). Source of truth for rates is the "Custom Loot Pool Allocation" Google Sheet. Ancient City map uses a datapack structure tag at `src/main/resources/data/famspecial/tags/worldgen/structure/on_ancient_city_maps.json` — add a sibling tag to support a new map destination. Per saved preference: do NOT add items to loot pools unless explicitly asked.
- **Client-only work (tooltips, HUD overlays, world-space rendering, client-side effects)** → `SYSTEMS.md` (`FamSpecialClient` + `HealthOverlay` sections). Tooltip state must mirror the server-side gate in `ArmorEffects` exactly — use the `grantLine(...)` / `buildSetLine(...)` / `grantLineWithCooldown(...)` helpers.
- **Multiplayer version-mismatch / handshake / login-phase networking** → `SYSTEMS.md` (`VersionHandshake` section). Strict equality policy; bump `mod_version` in `gradle.properties` for releases (`processResources` filters it into `fabric.mod.json`).
- **Necromancer zombie summon behavior (target lock, friendly fire, cleanup, cooldown, sound)** → `SYSTEMS.md` (`NecromancerSummon` section). If touching target-goal swap, also load `MIXINS.md` for `MobEntityAccessor`.
- **Poseidon dry-land riptide behavior** → `MIXINS.md` (`TridentItemMixin`) + `SYSTEMS.md` (`ArmorEffects.hasFullSet` helper).
- **Knight Radiant Shard durability (indestructible / regen / blocks-mending / broken state)** → `MIXINS.md` (the 5 mixins: `ItemStackMixin`, `ItemStackAttributeMixin`, `ItemStackEnchantmentsMixin`, `EnchantmentHelperMixin`, `EnchantmentHelperForEachMixin`) + `SYSTEMS.md` (`ArmorEffects` durability-regen block).
- **Thor set behavior (Mjolnir launch, Storm's Awakening, Asgardian's Flight, thunder sound)** → `ITEMS.md` (Thor section) + `SYSTEMS.md` (`MjolnirMaceItem` + `ThorEffects` sections + `ArmorEffects` Thor bonus entries) + `EFFECTS.md` (Asgardian's Flight / God of Thunder / Storm's Awakening rows). Thor is the only 5/5 set; `ASGARDIANS_FLIGHT` gates Mjolnir's launch via a status-effect predicate rather than re-deriving the set-check. Mjolnir ships with **Riptide V** (not cosmetic — `EnchantmentHelper.getTridentSpinAttackStrength` reads the level to scale launch magnitude). Storm's Awakening thunderstorm duration is **4800 t (~4 min)**; per-player cooldown is **24000 t (1 MC day)** stamped onto the worn Thunderhelm via the `STORM_COOLDOWN_END` component for tooltip MM:SS countdown. Storm Strider boots ship with **Feather Falling IV**.
- **Shurima set behavior (Sun Disc Spear KB-bypass, Sun's Protection helmet, Shuriman Endurance 4/4)** → `ITEMS.md` (Shurima section) + `SYSTEMS.md` (`ArmorEffects` Shurima bonus entries: `emperors_divide_mainhand`, `suns_protection_helmet_desert`, `shuriman_endurance_full_set_desert`) + `MIXINS.md` (`LivingEntityKnockbackMixin` for Emperor's Divide, `PlayerEntityExhaustionMixin` for Shuriman Endurance, `LivingEntityMixin` ModifyReturnValue #4 for Sun's Protection) + `EFFECTS.md` (Emperor's Divide / Sun's Protection / Shuriman Endurance rows). Both Sun's Protection (helmet-alone) and Shuriman Endurance (4/4) gate on a **desert-family biome** — `BiomeKeys.DESERT`, `BADLANDS`, or `ERODED_BADLANDS` (variable name `inDesert` is kept for continuity, semantic is "any desert-family biome"). `SHURIMAN_ENDURANCE` is the only **beacon-style** (non-`MOD_MANAGED`, 4800-t / 4-min, apply-once-gated) custom effect — its trigger drops on biome leave but the timer rides out naturally; do NOT add it to `MOD_MANAGED`. Sun's Protection is **not** in `MOD_MANAGED` — on helmet removal or biome leave, the trigger stops re-upping and the effect decays naturally over ~20 s. The equip-event `refreshBonusesFor` call means the re-up stops the same tick as the triggering event, but there is no explicit strip — the lingering damage reduction rides out its remaining duration.
- **Adding a new sound event** → `SYSTEMS.md` (`ModSounds` section). Register event → drop OGG under `assets/famspecial/sounds/` → add `sounds.json` entry.
- **Documentation reorg / CLAUDE.md updates** → edit `CLAUDE.md` + the relevant topic file. Keep `CLAUDE.md` minimal; push detail into topic files.
- **Bumping the mod version for a release** → `CLAUDE.md` (Commands section). Edit `gradle.properties` `mod_version`.
- **Don't know which file applies?** → read `INDEX.md` first; it maps every source file to its topic file.

## Loading discipline

- Do not load all topic files speculatively "in case they're relevant." Pick per this router; load more only if the first pass turns up a cross-reference you actually need.
- If a task crosses categories, load the narrowest pair first (e.g. `ITEMS.md` + `SYSTEMS.md`). Add `MIXINS.md` only if you're actually editing a mixin.
- `CLAUDE.md` "Working style" is binding for all tasks — respect the subagent / parallelism / background-execution triggers.
